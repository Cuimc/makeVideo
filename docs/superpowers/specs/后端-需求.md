# 二、后端开发 Prompt

```text
你是一名资深后端架构师和 NestJS 工程专家，请为我开发一个“新闻驱动的 AI 视频生成平台”的后端服务。请严格按照下面的业务需求、技术栈、数据结构、状态流转、积分规则和工程要求进行实现。目标是产出一个可直接联调的高质量后端工程，优先完成 MVP，不要过度设计。
```

# 一、项目目标

这是一个内容生产平台，核心业务流程如下：

1. 用户手机号验证码登录
2. 用户输入新闻关键词搜索新闻
3. 用户选择新闻后创建项目
4. 基于新闻和参考内容生成脚本
5. 基于脚本生成分镜
6. 用户编辑脚本和分镜并确认
7. 用户配置参数提交视频生成任务
8. 系统异步调用第三方视频生成接口
9. 视频结果进入成品库
10. 用户拥有积分账户，脚本生成、分镜生成、视频生成都要消耗积分
11. 后台需要采用冻结、扣除、返还机制，但前台不展示冻结积分概念

# 七、积分规则（后端必须严格实现）

## 1. 基本原则
- 所有需要消耗积分的动作，都要先校验余额
- 提交任务前先冻结预计积分
- 成功后正式扣除冻结积分
- 失败后返还冻结积分
- 部分成功时按实际成功结果扣除，多余冻结金额返还
- 所有动作都必须记录积分流水

## 2. 前台展示约束
前台不会展示冻结积分，但后台仍然必须保留冻结逻辑。
因此账户表中请至少保留：
- available_points
- frozen_points

## 3. 流水类型建议
请实现积分流水表，流水类型至少包括：
- RECHARGE
- FREEZE
- CONSUME
- REFUND

## 4. 任务维度
积分流水必须能关联具体业务对象，例如：
- 脚本生成任务
- 分镜生成任务
- 视频生成任务
- 充值订单

## 2. 统一响应结构
所有接口统一返回：

```ts
type ApiResponse<T> = {
  code: number
  message: string
  data: T | null
}
```

成功示例：
```js
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```
错误示例：
```js
{
  "code": 4001,
  "message": "积分不足",
  "data": null
}
```
